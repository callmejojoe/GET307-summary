Evaluation Metrics & Hyperparameter Tuning

Dr. I.M. Simeon, ABE Dept., FUTMINNA · GET 307 Course Materials
Section 1

Why Accuracy Alone Is Dangerous

Evaluation metrics quantify how well a machine learning model performs, especially on unseen data. The choice of metric is not trivial — it depends on the problem and the real-world consequences of different types of errors. Using the wrong metric can lead you to deploy a dangerously flawed model with full confidence.

⚠ The Accuracy Trap — A Critical Example

Imagine a fraud detection model in a dataset where 99% of transactions are legitimate and 1% are fraudulent. A model that simply predicts "Legitimate" for every single transaction achieves 99% accuracy — while catching exactly zero fraud cases. Accuracy here is a completely useless metric. This is the "accuracy paradox" on imbalanced datasets.

Section 2

Classification Evaluation Metrics

Before we can define these metrics, we need the Confusion Matrix — the foundation of all classification evaluation:

Predicted: Positive Predicted: Negative
Actual: Positive True Positive (TP) ✓ False Negative (FN) ✗ (missed it)
Actual: Negative False Positive (FP) ✗ (false alarm) True Negative (TN) ✓
📖 Accuracy

Accuracy = (TP + TN) / (TP + TN + FP + FN)

Proportion of all correct predictions. Simple and intuitive. Misleading for imbalanced datasets. Best used when classes are roughly equal in size.

📖 Precision — "When I say positive, am I right?"

Precision = TP / (TP + FP)

Of all the instances predicted as positive, what fraction actually are? High precision = few false alarms. Critical when the cost of a false positive is high (e.g., spam filter wrongly blocking important emails; unnecessary medical treatment).

📖 Recall (Sensitivity) — "Did I catch all positives?"

Recall = TP / (TP + FN)

Of all actual positives, what fraction did the model find? High recall = few missed positives. Critical when the cost of a false negative is high (e.g., cancer screening missing a true positive; fraud detection missing actual fraud).

🗞️ Analogy: The Net Fisherman

Precision: Of all the fish you pulled up in your net, what fraction were the type you wanted? High precision = few bycatch (unwanted fish caught by accident).

Recall: Of all the fish of your target type in the lake, what fraction did you catch? High recall = you missed very few target fish.

The trade-off: Use a finer net (higher recall) and you catch more target fish but also more bycatch (lower precision). Use a coarser net (higher precision) and every fish you catch is the right type — but many target fish slip through (lower recall).

📖 F1-Score — The Balanced Compromise

F1 = 2 × (Precision × Recall) / (Precision + Recall)

The harmonic mean of Precision and Recall. It's the go-to metric when both false positives and false negatives are costly and when you have imbalanced classes. A high F1 means the model has both high precision AND high recall — hard to achieve, but ideal.

📖 ROC–AUC: Overall Discriminative Power

The ROC (Receiver Operating Characteristic) curve plots True Positive Rate (Recall) against False Positive Rate at all possible classification thresholds. AUC (Area Under the Curve) summarizes the curve into a single number:

  • AUC = 1.0: Perfect model — correctly separates all classes at some threshold
  • AUC = 0.5: Random chance — no better than flipping a coin
  • AUC < 0.5: Worse than random (model has its labels backwards)

ROC-AUC is threshold-independent — it evaluates the model's overall ability to discriminate, not just at the default 0.5 cut-off.

Section 4

Hyperparameter Tuning: Controlling the Learning Process

In machine learning, hyperparameters are parameters set before training begins — they control how the learning process works, not what the model learns. The process of finding the best combination is called hyperparameter tuning or hyperparameter optimization.

Contrast with model parameters (e.g., weights in a neural network, coefficients in linear regression) — these are learned automatically during training. Hyperparameters must be set by the practitioner.

📖 Hyperparameter 1: Epochs

An epoch is one complete pass of the entire training dataset through the model. If you set epochs = 50, the model processes every training sample 50 times. More epochs generally = better learning (up to a point — after which overfitting begins). Too few = underfitting. Tune this by monitoring training vs. validation loss curves.

📖 Hyperparameter 2: Batch Size

A batch is the number of training samples processed before the model's parameters are updated. If you have 80 images and batch size = 16, you need 80/16 = 5 batches to complete one epoch. Smaller batches → noisier but more frequent updates. Larger batches → smoother, more stable updates. This rarely needs extensive tuning.

📖 Hyperparameter 3: Learning Rate

The learning rate controls how fast the model adjusts its parameters during training. Be very careful with this hyperparameter. Small learning rate → slow convergence, may get stuck. Large learning rate → fast but can overshoot the optimal solution and diverge. Typical values: 0.001–0.01. Often the single most impactful hyperparameter to tune.

🗞️ Analogy: Learning Rate as a Vehicle Throttle

Imagine you're driving to a precise GPS destination on a foggy mountain road. Learning rate is your throttle. Too slow (0.0001) and you'll barely move — you'll run out of time. Too fast (10.0) and you overshoot every turn and crash. The right learning rate (0.001–0.01) gets you there efficiently: fast enough to make progress, slow enough to adjust for each curve in the road.

📖 Hyperparameter 4: Number of Hidden Layers

In neural networks, more hidden layers = a more complex model with greater capacity to learn complex patterns. Too few layers → underfitting (too simple for the data). Too many → overfitting and much longer training times.

📖 Hyperparameter 5: Regularization Strength (λ)

Regularization adds a penalty to the loss function for model complexity, discouraging the model from overfitting. L1 (Lasso) can drive some weights to zero (sparse models). L2 (Ridge) shrinks all weights toward zero without eliminating them. Higher λ = stronger regularization = simpler model.

📖 Hyperparameter 6: Tree Depth (Decision Trees)

In decision trees, depth controls how many consecutive decisions are made. Shallow trees are interpretable but may underfit. Deep trees are flexible but easily overfit the training data. This is the kNN K value equivalent for tree models.

Section 5

Hyperparameter Tuning Techniques

TechniqueHow It WorksProsCons
Grid Search Try every combination of specified hyperparameter values Exhaustive — guaranteed to find the best combination within the grid Computationally expensive; scales exponentially with number of hyperparameters
Random Search Randomly sample combinations from specified distributions Faster than grid search; often finds near-optimal values efficiently Not exhaustive; may miss optimal combination
Bayesian Optimization Builds a probabilistic model of the objective function; targets promising regions Most sample-efficient; good for expensive evaluations More complex to implement; overhead from building the surrogate model

"Accuracy gives an overall correctness. Precision and recall focus on positive predictions. F1 balances both. ROC-AUC measures overall class separability."

— GET 307 Course Materials, Dr. I.M. Simeon

Metric Cheat Sheet

  • Accuracy: (TP+TN)/Total
  • Precision: TP/(TP+FP)
  • Recall: TP/(TP+FN)
  • F1: 2×P×R/(P+R)
  • AUC=1: Perfect model
  • AUC=0.5: Random guess
  • MAE: Mean|error|
  • RMSE: √MSE
  • R²: 1−SS_res/SS_tot

When to Use Which

  • Imbalanced classes → F1 or ROC-AUC
  • Cost of FP high → Precision
  • Cost of FN high → Recall
  • Regression → RMSE or R²
  • Outliers present → MAE (robust)

Hyperparameters

  • Epochs: Full passes over training data
  • Batch size: Samples per update step
  • Learning rate: Size of parameter updates
  • Hidden layers: Network depth/complexity
  • Regularization λ: Penalty for complexity
  • Tree depth: Decision tree complexity

Model vs. Hyperparameter

  • Model params → learned during training (weights, biases)
  • Hyperparams → set BEFORE training by practitioner

5 Questions · Select & Submit

Question 01

A disease screening model achieves 99% accuracy on a dataset where only 1% of patients have the disease. What does this tell us about the model's performance?

Correct: D. This is the "accuracy paradox" on imbalanced datasets. A model predicting "healthy" for every patient achieves 99% accuracy while catching zero disease cases (Recall = 0%). For medical diagnosis, Recall (sensitivity) is far more important — missing a true positive is dangerous.

Question 02

Precision measures which of the following?

Correct: B. Precision = TP / (TP + FP). It answers: "When the model predicts positive, how often is it right?" Option A describes Recall. Option C describes Accuracy. Precision is critical when false positives are costly (e.g., spam filters, legal fraud accusations).

Question 03

What is the F1-score, and why is it preferred over accuracy for imbalanced datasets?

Correct: C. F1 = 2 × (P × R) / (P + R). The harmonic mean is used (not arithmetic mean) because it gives lower weight to extreme values — a model with P=1.0 and R=0.001 gets F1 ≈ 0.002, not 0.5. This prevents a model from gaming the metric by being extremely conservative.

Question 04

In neural network training, what does the learning rate control?

Correct: A. The learning rate (α) scales the gradient in gradient descent: θ = θ − α × ∇L. Too large → overshoot the minimum → diverge. Too small → converge extremely slowly. Option B describes Epochs; Option C describes Batch Size.

Question 05

What is the key distinction between model parameters and hyperparameters in machine learning?

Correct: D. This is a fundamental distinction. Parameters (weights in a neural network, β coefficients in regression) are what training adjusts automatically. Hyperparameters (learning rate, number of layers, K in kNN, tree depth) must be specified by the practitioner and control how training works.
