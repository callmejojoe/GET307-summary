Classical ML Algorithms: KNN, Regression & Linear Models

Dr. I.M. Simeon, ABE Dept., FUTMINNA · GET 307 Course Materials
Section 1

Two Foundational Families of ML Models

In machine learning, models can be broadly grouped by how they learn patterns from data. Two foundational categories are distance-based models and linear models — the workhorses of classical machine learning, still widely used in engineering applications today.

Model Family Core Idea Main Example Best When
Distance-Based Predict based on similarity to known examples k-Nearest Neighbors (kNN) Small datasets, well-scaled features, no assumptions on data distribution
Linear Models Learn an explicit mathematical relationship between inputs and output Linear Regression, Logistic Regression Interpretability required, linear relationships exist in data
Section 2

k-Nearest Neighbors (kNN): The Neighbourhood Vote

The k-Nearest Neighbors algorithm is perhaps the most intuitive ML algorithm. It makes predictions by comparing a new (unseen) data point to existing examples in the training dataset using a distance measure. The prediction is determined by a "neighbourhood vote":

  • Classification: The new point is assigned the class most common among its K nearest neighbours (majority vote).
  • Regression: The new point is predicted as the average value of its K nearest neighbours.
🗞️ Analogy: Moving to a New Neighbourhood

Imagine you just moved to a new city and want to know if it's safe. You don't have a safety score — but you can look at the 5 houses nearest to yours. If 4 out of 5 are well-maintained with gardens and good lighting, you conclude it's a safe neighbourhood. That's kNN: you're classified by the majority vote of your K nearest neighbours (K=5 here). Change K to 1, and you'd be classified by the single nearest house — potentially misleading.

Section 3

Steps to Implement kNN and Calculate K-Distance

  1. Define K

    Choose the number of nearest neighbours to consider. K is a hyperparameter — you set it before training. Small K = more complex, noise-sensitive model. Large K = smoother, less flexible model.

  2. Calculate Distances

    Compute the distance between the input query point and every point in the training dataset using one of the distance metrics below.

  3. Sort Distances

    Arrange the calculated distances in ascending order (closest first).

  4. Identify the K-th Distance

    The K-distance is the distance value at the K-th position in the sorted list. All points with distance ≤ this K-th distance are considered neighbours.

  5. Select Neighbours and Predict

    Use the K selected neighbours to make a prediction via majority vote (classification) or average value (regression).

Section 4

Distance Metrics: How "Closeness" Is Measured

The definition of "nearest" depends entirely on the distance metric you choose. The three most common metrics in kNN are:

📖 1. Euclidean Distance (L2 Norm) — Most Common

d(x, y) = √ Σⱼ (xⱼ − yⱼ)²

The straight-line ("as the crow flies") distance between two points in d-dimensional space. For 2D: √((x₁−y₁)² + (x₂−y₂)²). This is Pythagoras' theorem extended to multiple dimensions.

📖 2. Manhattan Distance (L1 Norm) — Grid-Based

d(x, y) = Σⱼ |xⱼ − yⱼ|

Distance measured along axis-aligned paths — like navigating city blocks. You can only move horizontally or vertically, never diagonally. Useful when features represent independent discrete steps.

📖 3. Minkowski Distance — The General Form

d(x, y) = (Σⱼ |xⱼ − yⱼ|ᵖ)^(1/p)

A generalisation: when p=2, it equals Euclidean distance. When p=1, it equals Manhattan distance. By tuning p, you control the sensitivity to large differences in any single dimension.

🗞️ Analogy: Crow vs. Taxi

Euclidean distance is how a crow measures distance — straight line, no obstacles. Manhattan distance is how a Lagos taxi measures distance — you can't drive through buildings, you take the grid of streets. For data in a structured grid (like pixels in an image, or grid-based sensor readings), Manhattan distance is more naturally meaningful. For continuous, multi-dimensional data, Euclidean is often the better default.

Section 5

Worked Numerical Example: K = 3

Suppose a query point has the following computed distances to five training points:

Training PointDistance to Query
Point A2.5
Point B1.2
Point C3.8
Point D0.5
Point E4.2

Step 1 — Sort ascending: [0.5, 1.2, 2.5, 3.8, 4.2]

Step 2 — K = 3, so K-distance = 2.5 (the 3rd value in sorted list)

Step 3 — Neighbours: Points D (0.5), B (1.2), and A (2.5) are selected.

Step 4 — Predict: If these 3 neighbours have classes [Positive, Positive, Negative] → majority vote = Positive.

Section 6

Python Implementation with scikit-learn

The following code demonstrates kNN using scikit-learn's NearestNeighbors class:

import numpy as np
from sklearn.neighbors import NearestNeighbors

# Sample training data (5 points in 2D space)
X = [[1, 2], [2, 3], [3, 4], [6, 7], [7, 8]]

# Query point — we want to classify/predict for this
query_point = [[4, 5]]
k = 3

# Fit the NearestNeighbors model
nn = NearestNeighbors(n_neighbors=k)
nn.fit(X)

# Find the K nearest neighbours and their distances
distances, indices = nn.kneighbors(query_point)

print("Distances:", distances[0])
# Output: distances to the 3 nearest neighbours

print("K-distance (3rd nearest):", distances[0][-1])
# Output: the distance to the K-th nearest neighbour

print("Neighbour indices:", indices[0])
# Output: indices into X of the K nearest neighbours
⚠ Critical Gotcha: Feature Scaling

kNN is extremely sensitive to feature scaling. If Feature A ranges from 0 to 1 and Feature B ranges from 0 to 10,000, distance calculations will be dominated entirely by Feature B. Always normalise or standardise features before using kNN. Use StandardScaler or MinMaxScaler from scikit-learn.

Section 7

Linear Models: Learning Mathematical Relationships

Linear models assume a linear relationship between input features and the output. They are the simplest, most interpretable family of ML models and often serve as the baseline against which more complex models are benchmarked.

📖 Linear Regression — For Continuous Outputs

Linear regression predicts a continuous outcome by fitting a straight line (or hyperplane in multiple dimensions) through the data that minimises prediction error (specifically, minimises the sum of squared residuals).

ŷ = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ

Where β₀ is the intercept, β₁...βₙ are learned coefficients, and x₁...xₙ are input features. Training finds the β values that minimise MSE.

📖 Logistic Regression — For Classification (Despite the Name!)

Despite its name, logistic regression is a classification algorithm, not a regression algorithm. It models the probability that an input belongs to a particular class using a sigmoid (logistic) function, then assigns a class based on a probability threshold (typically 0.5).

P(y=1 | x) = σ(β₀ + β₁x₁ + ... + βₙxₙ)

Where σ(z) = 1 / (1 + e^(−z)) is the sigmoid function. Output is always between 0 and 1 — a probability. Prediction: if P > 0.5 → Class 1; if P ≤ 0.5 → Class 0.

🗞️ Analogy: The Exam Score vs. Pass/Fail Decision

Linear regression is like predicting a student's exam score (a number on a continuous scale from 0 to 100) based on hours studied.

Logistic regression is like deciding whether the student passes or fails — a binary outcome. It asks: "Given the hours studied, what is the probability of passing?" If P(pass) > 0.5, predict "Pass." The sigmoid function squashes any linear combination into a [0,1] probability range — which is much more sensible for a yes/no output than a raw linear number.

Section 8

Summary: When to Use Which Algorithm

AlgorithmProblem TypeProsCons
kNN Classification & Regression Simple, no training phase, intuitive Slow on large datasets, sensitive to feature scaling and irrelevant features
Linear Regression Regression only Interpretable, fast, works well on linear data Fails on non-linear relationships, sensitive to outliers
Logistic Regression Binary/multi-class Classification Interpretable, probabilistic output, efficient Assumes linear decision boundary — fails on complex, non-linear problems

Key Algorithms

  • kNN: Distance-based, no training, lazy learner
  • Linear Regression: Continuous output, minimises MSE
  • Logistic Regression: Binary classification via sigmoid
  • Others: Decision Trees, SVM, Random Forest, Clustering

Distance Metrics

  • Euclidean (L2): Straight-line distance
  • Manhattan (L1): Grid/city-block distance
  • Minkowski: General form (p=1 or p=2)
  • ⚠ Always scale features before using distance metrics!

Key Formulas

  • Euclidean: √Σ(xⱼ−yⱼ)²
  • Manhattan: Σ|xⱼ−yⱼ|
  • Linear Reg: ŷ = β₀ + β₁x₁ + ...
  • Sigmoid: σ(z) = 1/(1+e^−z)
  • Logistic: P(y=1|x) = σ(βᵀx)

The K Hyperparameter

  • Small K → complex, overfits to noise
  • Large K → smoother, may underfit
  • Odd K avoids ties in binary classification
  • Typical starting points: K=3, 5, 7

5 Questions · Select & Submit

Question 01

In the k-Nearest Neighbors algorithm, how is the prediction determined for a classification task?

Correct: B. kNN for classification uses a majority vote of the K closest training examples. No model is trained — predictions are made at inference time by computing distances. This is why kNN is called a "lazy learner."

Question 02

A query point has distances to 5 training points: [3.1, 0.8, 5.5, 1.4, 2.3]. With K=3, what is the K-distance and which points are neighbours?

Correct: C. Sorted distances: [0.8, 1.4, 2.3, 3.1, 5.5]. The 3rd value (K=3) is 2.3 — this is the K-distance. The 3 nearest neighbours are those with distances 0.8, 1.4, and 2.3 (all ≤ 2.3).

Question 03

Why is feature scaling (normalisation/standardisation) critical before applying kNN?

Correct: D. If Feature A is 0–1 and Feature B is 0–10,000, distances are almost entirely determined by Feature B. Scaling (e.g., StandardScaler) puts all features on comparable ranges, so each contributes fairly to distance calculations.

Question 04

Despite its name, Logistic Regression is used for which type of machine learning task?

Correct: A. Despite "regression" in its name, Logistic Regression is a classification algorithm. It uses the sigmoid function to output a probability (0 to 1), then assigns a class based on a threshold (typically 0.5). It's a common exam trick question!

Question 05

What is the key difference between Euclidean and Manhattan distance?

Correct: B. Euclidean distance is the "crow flies" straight-line distance (involves square root). Manhattan distance sums absolute differences along each axis — like navigating a grid city where you can only move along streets, not diagonally. Both work in any number of dimensions.
